Резюме

Все 47 страниц академии имеют два дефекта на уровне шаблона, которые активно подавляют позиции: весь сайт объявляет lang="en", хотя таргетирует русскоязычных пользователей, а каждая страница курса отдаёт заголовок CTA-виджета («Сомневаетесь в выборе?») как единственный H1 — само название курса размечено дивом. Оба дефекта исправляются одним изменением шаблона и дают немедленный эффект.

Метаданные контента (title, description) непоследовательны и бедны ключевыми словами: в большинстве заголовков нет слов «курс» и «онлайн», а описания не содержат инструментов, длительности и социальных доказательств — именно того, что определяет CTR в выдаче по курсам. Структурированные данные отсутствуют на всех 46 страницах курсов.

  • 2Повысить div с названием курса до <h1> в шаблоне курса (все 46 страниц)
  • 3Понизить CTA <h1> → <h2>, чтобы на странице был только один H1 (все 46 страниц)

Критично

Изменить в шаблоне базового лейаута:

<html lang="en"> → <html lang="ru">

Единственный H1-сигнал Google на каждой странице курса — «Сомневаетесь в выборе?» (CTA-форма). Название курса — важнейший ключевой сигнал на странице — невидимо для краулера.

Исправление

  • <div class="section__title _h2"> → <h2 class="section__title">
  • <h1 class="somnevaetes__form-head-title"> → <h2>

#3 Название курса в <title> ≠ видимый заголовок
Критично
Доказательство (страница DevOps)
<title>DevOps-инженер с нуля</title> <div class="courseIntro-title">Стань инженером Devops с нуля</div>
Влияние

Разный порядок слов, разная капитализация (DevOps vs Devops), разный поисковый интент. Google согласовывает это, но смешанные сигналы снижают уверенность в целевом ключевом слове страницы.

Исправление

Привести <title>, <h1> и og:title к одной строке. Формат <title> лучше для SEO (роль первой), стандартизировать H1 под него.

#7 Нет структурированных данных Course (JSON-LD)
Высокий/span>
Доказательство

Ноль JSON-LD на любой странице курса. Google поддерживает схемы Course и CourseInstance для онлайн-образования — может отображать детали курса (провайдер, цена, длительность) в расширенных результатах.

Исправление — добавить JSON-LD к шаблону курса
{ "@context": "https://schema.org", "@type": "Course", "name": "DevOps-инженер с нуля", "description": "Станешь DevOps-инженером за 6 месяцев...", "provider": { "@type": "Organization", "name": "Merion Academy", "sameAs": "https://wiki.merionet.ru/merion-academy" }, "hasCourseInstance": { "@type": "CourseInstance", "courseMode": "online", "inLanguage": "ru" } }

Заполнять динамически из базы данных. Одно изменение шаблона → 46 страниц получают схему.

#9 Нет хлебных крошек на страницах курсов
Средний/span>

Страницы статей показывают хлебные крошки типа «VoIP › Как настроить»; страницы курсов — ничего. Хлебные крошки в SERP (через схему) увеличивают кликабельный размер и помогают пользователям понять место страницы в иерархии.

Исправление

Добавить UI хлебных крошек + BreadcrumbList JSON-LD: Merion Academy → [Категория] → [Название курса]

#10 Имена инструкторов есть на странице, но нет в мета или схеме
Средний/span>

Страницы курсов показывают имена инструкторов (например, «Филипп Игнатенко» в DevOps). Это актив E-E-A-T. Но нет тега author в мета, нет ссылки на страницу профиля, нет схемы Person, связывающей инструктора с контентом.

Исправление

Добавить <meta name="author" content="Филипп Игнатенко">, создать страницы профилей /authors/[имя] и добавить поле instructor в схему Course.

#11 Нет тегов hreflang
Средний/span>

Ни один тег hreflang не найден ни на одной странице. Поскольку контент ориентирован на несколько русскоязычных рынков (RU, KZ, BY, UZ, KG), самоссылающиеся теги hreflang="ru" укрепляют языковой таргетинг.

Исправление

Добавить самоссылающиеся теги hreflang="ru" на все 47 страниц.
